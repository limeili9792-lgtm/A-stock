# A股个股三层递进分析

判断"公司好不好"。三层递进，不能停在任何一层就下结论。**交易面判断走 `stock-trading` 技能，本技能不输出买卖建议。**

## 核心原则

- **数据先行** — 不拉数据不做结论，数据不足直接说明，禁止推测
- **统一数据源** — 所有数据引用以 `../../references/数据字典.md` 为唯一索引，优先主源→兜底，不超范围
- 同一CSI四级分类下的公司商业模式可能完全不同，不做PE直接横向对比
- ROE必须杜邦拆解，结论基于ROA而非ROE
- 每层有明确的"不可跳过"步骤（标注⚠️）

### 数据引用流程

```
1. 确定分析步骤 → 2. 查数据字典找到主源 → 3. 主源可用？
   ├─ ✅ → 拉数据（标记置信度：高）
   └─ ❌ → 兜底WebSearch（标记置信度：中）
        └─ 兜底也不可用 → 标注"数据不足"，不做结论，禁止推测补位
```

## 触发条件

用户问"分析下XX""怎么看XX""是什么公司""基本面"等。输出结构：结论先行→三层递进→自反检查。

## 第一层：定量初筛 + 行业位置

### 1.1 行业位置（先于数据看）

- 公司在产业链什么位置？上游/中游/下游？
- 行业是成长型、周期型、还是稳定型？
- 行业处于什么阶段（导入/成长/成熟/衰退）？
- 核心驱动力是什么（政策？技术？价格？）？

目的：建立坐标系。20%增速在成熟行业是复苏信号，在成长行业是掉队信号。

### ⚠️ 1.1.1 同业头部扫描（不可跳过）

1. 查 `references/中证行业分类.csv`（随技能打包，覆盖5512只A股，四级分类链路），输出完整四级分类链：`一级(XX) → 二级(XXXX) → 三级(XXXXXX) → 四级(XXXXXXXX)`
2. 输出：`四级分类XXX，共X家公司 | 个股 市值第Y / 营收第Z`
3. 拉出该分类全部公司，按市值降序取Top10，产出表格：

```
| 排名 | 代码 | 简称 | 市值 | 营收 | 净利率 | PE_TTM | 真正同业？|
```

4. 从Top10标注真正可比同业，筛除"共用一个标签但做不同事"的。判断依据：
   - 收入结构是否重叠？
   - 客户群是否重叠？
   - 技术路线是否可比？

### ⚠️ 1.1.2 同业质量检查（不可跳过）

```
□ 真正同业≥2家？
□ 毛利率离散度合理？（max-min<30pp合理，>50pp分类混杂）
□ 收入结构真的重叠？
```

任一项不通过→标记"非真正同业分类"，不强行做PE对比。改为跨分类找业务对标，**只做业务维度对比（毛利率/增速/技术），不做PE直接比较**。

### 1.2 定量筛选 + 杜邦拆解

> 数据源：`../../scripts/akshare_utils.py` → `get_financial_summary(ts_code)` 一键提取
> 兜底：WebSearch 搜年报/季报

**⚠️ ROE必做杜邦拆解：**
- 权益乘数 = 1 / (1 - 资产负债率)
- ROA = ROE / 权益乘数
- 交叉验证：ROE ≈ 净利率 × 周转率 × 权益乘数

横向对比ROA/净利率/周转率/权益乘数。**结论基于ROA而非ROE。**

常见陷阱：同一CSI四级分类商业模式可能完全不同；ROE高可能是杠杆推的；利润暴增可能只是基数低。

产出：完整分类链 + 同业Top10排名 + 杜邦对比表。

## 第二层：业务拆解

### 2.1 收入结构

> 数据源：`../../scripts/extract_annual_report.py` → `extract_section(code, year, "revenue_breakdown")`，自动 pdfplumber 提取 → 评分<70 时 Vision 回退。也可用 WebSearch 兜底。

```
| 业务板块 | 营收占比 | 毛利率 | 利润贡献占比 | 同比变化 | 备注 |
```

### 2.2 技术路线 vs 同行

```
| 维度 | 个股 | 同业A | 同业B |
|:---|------|------|------|
| 技术路线 | ... | ... | ... |
| 核心原材料自给率 | ... | ... | ... |
| 产品结构差异 | ... | ... | ... |
```

### 2.3 客户结构

前五大>60%、最大客户>30%为不健康。

### ⚠️ 2.4 反方证据强制检查（不可跳过）

**强制输出至少2条反方论点**。偏正面找负面信号，偏负面找正面信号。不是找新数据，是重新审视已有数据中被忽略的信号。

```
当前结论倾向：看空 / 看多 / 中性
反方论点1：[用已有数据支持]
反方论点2：[用已有数据支持]
这些反方论点是否足以推翻主结论？是/否
```

给不出2条以上有效反方论点→存在确认偏误，退回补充数据。

### 2.5 最新季度利润质量（有最新季报时必做）

**步骤一：净利率桥接** `净利率 = 毛利率 - 费用率 + 非主营比重 + 其他`

对比本季度 vs 去年同期，缺口变化揭示利润驱动力。

**步骤二：CFO/净利润趋势** 连续季度为负→利润纸面化。

**步骤三：增速一致性** 营收↑ + 毛利率↑/→ + 净利↑ = 健康；营收↓ + 毛利率↓ + 净利↑ = 非经营驱动。

**步骤四：季节性校验** Q1/全年占比反推全年利润。

产出：利润质量判断（健康/黄色预警/红色预警）+ 驱动力归因。

## 第三层：风险定价

### 3.1 结构性风险

```
| 风险项 | 触发条件 | 影响程度 | 时间窗口 | 可逆？|
```

### 3.2 资产负债表质量

```
| 检查项 | 数值 | 警戒线 | 结论 |
| 对外担保/净资产 | X% | >30% |
| 大股东质押比例 | X% | >50% |
| (存货+应收)/营收 | X% | 同比大幅上升 |
```

### 3.3 周期位置

| 指标 | 当前 | 12月前 | 24月前 | 方向 | 驱动因素 |

### 3.4 估值判断

不是看PE绝对值，看市场在定价什么故事：
```
市场在定价：____________________
我认为市场错了什么：____________________
反转触发条件：____________________
```

### ⚠️ 3.5 自反检查（输出前最后一步，不可跳过）

真诚拆自己的台。≥3条可质疑之处，每条说明严重程度和对主结论的影响。检查维度：推论跳跃、数据盲区、时间对齐、对比公平。如果自反发现主结论支撑逻辑有严重漏洞，修正后再输出。

## 纪律

1. 不能因为ROE高就说好，先拆杠杆，结论基于ROA
2. 不能因为同一行业分类就以为可比
3. 不能只看利润表不看资产负债表
4. 不能因为"会被颠覆"就否定，关键看时间错配
5. **禁止跨行业/跨个股对比**——每次分析从零开始
6. 不在本技能中输出买卖建议，交易面走 `stock-trading`
